Fynetic Web Server Documentation

Introduction

The Fynetic Web Server is a lightweight, fast, and simple local server that allows you to host static content (like HTML, CSS, and JavaScript) on your local machine. It's perfect for developers and small projects, designed for ease of use with minimal configuration.

How to Set Up the Fynetic Web Server

Follow these steps to set up the Fynetic Web Server on your local machine:

  1. Download the Installer: Download the installer by clicking the button below.
  2. Run the Installer: After downloading the installer, double-click the fynetic-installer.bat file to install fce.exe to C:/fce. The fce.exe file is the main executable that runs the server.
  3. Set Up Web Server Files: The web server files need to be set up in the directory where you want the server to run. Open the Command Prompt, navigate to the desired directory, and run the following command: fce setup
    This command will set up the necessary web server files in the directory.

  4. Start the Server: Once the setup is complete, start the server by running the following command in the same directory: fce server -p 8080

    By default, the server will run on port 8080. You can change the port by using the -p flag (e.g., fce server -p 7070 to run it on port 7070).

Available Commands

fce

Displays the current version of the Fynetic Web Server. Useful to ensure you're using the latest version.

fce server

Starts the web server. Use the -p flag to specify a custom port (e.g., fce server -p 8081).

fce setup

Sets up the necessary files for the web server in the directory where the command is run. Run this command after downloading fce.exe to configure the server.

Server Configuration

Fynetic Web Server comes with a few configurable options:

  • Port: You can configure the port for the server using the -p flag when running fce server.
  • Directory: The server serves files from the directory where it was initialized. You can change the directory by running fce setup in a different location.
  • Logging Level: You can configure the log level by modifying the config file in the server directory.

Troubleshooting

If you run into issues, here are some common problems and solutions:

  • Server Not Starting: Ensure the port isn't already being used by another service. You can specify a different port using the -p flag.
  • Permissions Issues: Make sure you have the necessary permissions to run commands in the directory where you're setting up the server.
  • File Not Found: Ensure that the directory you're serving contains the correct files. Double-check the file paths to avoid errors.
