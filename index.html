Fynetic Web Server Documentation

Introduction

The Fynetic Web Server is a lightweight, efficient local server that makes hosting simple and fast for developers and small projects. It’s designed with minimalism in mind, enabling you to host static content like HTML, CSS, and JavaScript files with ease on your local machine. Fynetic Web Server aims to be the simplest solution for personal development servers.

How to Set Up the Web Server

Follow the steps below to set up the Fynetic Web Server on your local machine:

  1. Download the web server files: Place the Fynetic Web Server files in an empty directory on your local machine. You can download the necessary files from our GitHub repository.
  2. Run the setup: Open the terminal and navigate to the directory where you placed the files. Run the command fce setup to initialize the directory structure and install the required dependencies.
  3. Start the server: Once the setup is complete, run fce server to start the web server. By default, the server will run on port 8080. You can change the port by using the -p flag, e.g., fce server -p 8081 to run it on port 8081.

Available Commands

fce

This command shows the current version of the Fynetic Web Server. It’s useful to check if you’re running the latest version.

fce -u

Use this command to simulate an update of the Fynetic Web Server. It will download the latest version and display a progress bar. This is useful for testing the update process.

fce server

This starts the web server. If you want to specify a different port, use the -p flag (e.g., fce server -p 8081 to run the server on port 8081).

fce setup

Runs the setup process, which initializes the web server’s directories and installs any dependencies.

Server Configuration

Fynetic Web Server comes with some basic configuration options that you can adjust based on your needs:

  • Port: You can configure the port the server runs on by specifying the -p flag during the fce server command.
  • Directory: The server serves files from the directory where it was initialized. You can customize this by modifying the setup.
  • Log Level: You can change the logging level by modifying the config file to debug or error logging based on your needs.

Troubleshooting

If you encounter any issues while running the Fynetic Web Server, here are some common problems and their solutions:

  • Server Not Starting: Ensure that no other services are running on the same port. Try specifying a different port with the -p flag.
  • Permissions Issues: If you're on a restricted system, make sure you have the necessary permissions to run commands in the directory.
  • File Not Found: Make sure that the directory you are serving contains the files you intend to serve. Double-check the file paths.
